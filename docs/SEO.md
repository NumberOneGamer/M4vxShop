# SEO

## Metadata Strategy

Every page uses the `createMetadata()` helper from `src/lib/metadata.ts`:

```typescript
import { createMetadata } from "@/lib/metadata"

export const metadata = createMetadata({
  title: "Shop",
  description: "Browse our curated collection.",
  path: "/shop",
})
```

### Default Values

| Field | Value |
|-------|-------|
| Site name | M4vx |
| Default title | M4vx \| Premium Dropshipping |
| Default description | Premium products curated for modern lifestyles. |
| Locale | en_US |
| OG type | website |

### Per-Page Overrides

- `title` — Page title (appended with "| M4vx")
- `description` — Page-specific meta description
- `path` — Canonical URL path
- `ogImage` — Custom Open Graph image URL
- `noIndex` — Set `true` to prevent indexing (admin pages, auth pages)

## Structured Data

### Organization Schema

Homepage includes `OrganizationSchema` component (`src/components/shared/seo/organization-schema.tsx`) with:
- `name`: M4vx
- `url`: Site URL
- `logo`: Logo URL
- `description`: Site description

### Product Schema

Product pages include `ProductSchema` with:
- `name`, `description`, `image`
- `sku`, `brand`
- `offers` (price, currency, availability)

### BreadcrumbList

Breadcrumb navigation renders `BreadcrumbList` structured data.

## Open Graph Images

Default OG image is generated by Vercel OG. Custom OG images can be passed to `createMetadata({ ogImage: "..." })`.

OG tags are set on:
- Homepage
- Product pages (with product image)
- Category pages
- Shipping/Returns

## Sitemap

| Entry | Frequency | Priority |
|-------|-----------|----------|
| Homepage | weekly | 1.0 |
| Shop | daily | 0.8 |
| Product pages | weekly | 0.6 |
| Category pages | weekly | 0.6 |

## Robots

```typescript
// Default - index and follow
robots: { index: true, follow: true }

// Admin/auth - noindex
robots: { index: false, follow: false }
```

## Canonical URLs

All pages include `<link rel="canonical" href="...">` via the `createMetadata()` helper or page-level `alternates`.

## Performance Considerations

- Fonts use `display: swap` to prevent invisible text (FOIT)
- Images use Next.js `<Image>` with lazy loading
- API responses cached with `stale-while-revalidate`
- Static pages use `revalidate = 3600` (ISR)
